# git提交

当用户需要用git提交代码时使用此skill。

## 使用场景

- 用户要求提交代码更改
- 用户需要记录代码变更历史

## 操作步骤

1. **查看文件状态**
   ```bash
   git status
   ```
   查看哪些文件被修改或新增
2. **查看差异内容**
   ```bash
   git diff
   ```
   查看具体的代码变更内容
3. **提交更改**
   ```bash
   git add .
   git commit -m "清晰描述更改内容"
   ```
4. **推送更改**
- 仅在用户明确要求推送时执行!!!
   ```bash
   git push origin main
   ```

## 注意事项

- 提交信息应简洁明了，不必要逐文件解读，描述清楚主要更改内容即可。
- 如果感到困惑，可以向用户发起询问。
