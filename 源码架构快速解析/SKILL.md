# 源码架构快速解析
name: source-code-architecture-analysis
description: 专为新人设计的项目源码解析工具，生成易懂的架构解读文档，包括目录结构、核心模块、功能 - 代码映射、快速上手路径
version: 2.0
author: Trae User

## 核心指令（智能体执行步骤）
instructions:
  ### 步骤 1：全局概览
  1.1 读取项目根目录的 README.md、CONTRIBUTING.md、ARCHITECTURE.md（如有），提取核心信息：
       - 项目定位：这个项目是做什么的？（用一句话说清楚）
       - 技术栈：使用了哪些语言、框架、关键依赖？
       - 官方架构说明：是否有官方标注的架构图或设计文档？
  
  1.2 遍历并列出项目核心目录结构（忽略 node_modules、dist、.git、build 等构建/依赖目录）：
       - 按功能分类标注：如 `src/` → 核心源码 | `test/` → 测试用例 | `docs/` → 文档 | `packages/` → 子包
       - 只关注核心目录（不超过 10 个），避免信息过载
  
  1.3 识别项目入口文件（如 index.js、main.py、App.vue），用一句话标注其作用。

  ### 步骤 2：模块拆解（理解核心代码组织）
  2.1 对每个核心目录（优先 src、lib、core 等），分析：
       - 列出关键文件（按代码行数和重要性排序，每个目录不超过 5 个文件）
       - 每个文件的核心功能（用一句话说清楚，避免技术术语）
       - 文件间的依赖关系（如 A 文件导入 B 文件的核心函数，用简单箭头表示）
  
  2.2 识别核心业务模块（如认证模块、数据处理模块、UI 组件模块），标注：
       - 模块功能：这个模块负责什么？
       - 核心文件：该模块的关键文件路径
       - 核心逻辑：输入什么 → 处理什么 → 输出什么（用流程图或简单描述）

  ### 步骤 3：功能 - 代码映射
  3.1 从 README 或文档中提取「官方宣称的核心功能」列表（如「支持批量导出数据」「提供可视化图表」）
  
  3.2 为每个功能点，精准定位对应的代码实现位置：
       - 功能名称 → 所属模块 → 核心文件 → 关键函数/类名
       - 示例：「用户登录」→ `src/auth/` → `login.js` → `validateUser()` 函数
  
  3.3 标注功能的扩展点（如钩子函数、配置项、插件接口），方便后续二次开发。

  ### 步骤 4：新人友好信息补充
  4.1 识别「新手友好模块」：
       - 代码结构简单、注释完善的文件
       - 独立性强、依赖少的功能模块
       - 适合练手的 bug 修复或功能增强点
  
  4.2 标注「核心高频修改文件」：
       - 通过 git log 分析（如有）或代码结构推断
       - 标注哪些文件最常被修改，新人需要重点关注
  
  4.3 提示「易出错的关键逻辑」：
       - 复杂的状态管理
       - 异步处理逻辑
       - 边界条件处理
  
  4.4 提供「快速上手路径」：
       - 推荐新人优先阅读的 3-5 个核心文件
       - 建议的代码阅读顺序
       - 本地运行/调试的步骤（从文档或 package.json/scripts 等提取）

  ### 步骤 5：输出结构化解读文档
  5.1 输出格式要求（必须按此结构，清晰易读）：
       - 标题：[项目名] 源码架构解读
       - 必须包含以下章节：
         1. 项目全局信息（一句话定位 + 技术栈）
         2. 核心目录/文件功能说明（表格形式）
         3. 核心业务模块解析
         4. 功能 - 代码映射表（表格形式）
         5. 新人快速上手指南（重点章节）
  
  5.2 语言要求：
       - 必须使用中文
       - 每个技术概念首次出现时，用括号简单解释
  
  5.3 重点标注（使用 emoji 或标记）：
       - 🟢 新手友好模块
       - 🔴 核心高频修改文件
       - ⚠️ 易出错的关键逻辑
       - 💡 推荐优先阅读

## 依赖工具（需绑定的 MCP）
tools:
  - filesystem.read  # 读取文件内容（核心）
  - filesystem.list  # 遍历目录结构
  - terminal.run     # 可选：执行 git log、tree、find 等命令辅助分析
  - code-search      # 可选：搜索关键函数/类名、引用关系

## 提示
- 如果源码仓库过大 (目录很多)，优先查看核心部分代码，如 src、core、lib 等目录
- 对于每个章节，优先保证质量而非数量，每个部分不超过 5-7 个条目
- 遇到不懂的技术术语，要在文档中用括号简单解释
- 多使用 emoji 和标记，让文档更生动易读
- 始终站在新人角度思考：「如果我是第一次接触这个代码库，我最想知道什么？」
